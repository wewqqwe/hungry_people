# Hungry People

Сайт ресторана на PHP.

Проект включает адаптивный лендинг, регистрацию и авторизацию пользователей, восстановление пароля, бронирование столиков, форму обратной связи и хранение контента разделов в базе данных.

## Возможности

### Пользовательская часть

- адаптивная одностраничная верстка;
- фиксированная шапка с уменьшением логотипа при прокрутке;
- якорные ссылки и мобильное меню;
- фильтрация блюд по категориям;
- галерея, слайдер специалитетов и Яндекс Карта;
- регистрация, вход и выход без перезагрузки;
- восстановление пароля через email;
- бронирование столиков;
- форма обратной связи.

### Серверная часть

- MVC: модели, контроллеры, точка входа `ajax.php`;
- автозагрузка классов через composer;
- работа с базой через PDO и подготовленные выражения;
- хранение паролей в виде хеша;
- сессии для авторизации, состояние сохраняется после перезагрузки;
- вывод разделов из базы: блоки about и team, специалитеты, меню, события;
- вывод на главной до 21 блюда по флагу `on_main`;
- восстановление пароля по одноразовому токену;
- отправка писем о бронировании и восстановлении пароля.

## Стек

- PHP 8;
- MySQL;
- HTML, CSS, JavaScript;
- jQuery;
- Slick Carousel;
- Bootstrap 4;
- Composer.

## Установка

### 1. Размещение проекта

Скопируйте папку в каталог сайтов локального сервера:

```text
OSPanel      C:\OSPanel\home\
OpenServer   domains\
XAMPP        htdocs\
```

### 2. База данных

Импортируйте `install.sql` через phpMyAdmin: Импорт → выберите файл → Вперёд.

Скрипт создаст базу `hungrypeople`, таблицы и добавит начальный контент сайта. Запускать можно повторно — таблицы пересоздаются заново.

### 3. Подключение к базе

Параметры указаны в `db.php`:

```php
foreach (array('localhost', 'MySQL-8.4') as $host) {
    $pdo = new PDO('mysql:host=' . $host . ';dbname=hungrypeople;charset=utf8', 'root', '');
}
```

Хосты перебираются по очереди. `localhost` подходит для XAMPP и OpenServer, `MySQL-8.4` — имя модуля базы в OSPanel, который не держит MySQL на localhost.

### 4. Автозагрузка классов

Папка `vendor` уже в проекте. Если её нет:

```bash
composer dump-autoload
```

### 5. Запуск

Откройте сайт по адресу локального сервера:

```text
http://hungrypeople.local/
```

Сайт должен открываться через веб-сервер. По адресу `file:///` PHP не выполняется и формы не работают.

## Настройка почты

Письма отправляются функцией `mail()`. На локальном сервере она не работает без посредника: под Windows `mail()` не поддерживает авторизацию и TLS, а Gmail без них соединение не принимает.

### Вариант с XAMPP

В `C:\xampp\php\php.ini`:

```ini
SMTP = smtp.gmail.com
smtp_port = 587
sendmail_from = ваш-адрес@gmail.com
sendmail_path = "C:\xampp\sendmail\sendmail.exe -t"
```

В `C:\xampp\sendmail\sendmail.ini`:

```ini
smtp_server=smtp.gmail.com
smtp_port=587
auth_username=ваш-адрес@gmail.com
auth_password=пароль-приложения
```

### Вариант с OSPanel

Sendmail отсутствует, вместо него используется модуль Mailpit. Включите его в меню OSPanel (Модули) и перезапустите.

Письма открываются в браузере:

```text
https://mailpit
```

Для отправки на настоящий адрес в `C:\OSPanel\config\Mailpit\default\settings.ini` включите пересылку:

```ini
MP_SMTP_RELAY_ALL    = true
MP_SMTP_RELAY_CONFIG = {root_dir}\modules\{module_name}\config\MP_SMTP_RELAY_CONFIG.txt
```

и заполните `C:\OSPanel\config\Mailpit\default\templates\MP_SMTP_RELAY_CONFIG.txt`:

```yaml
host:      smtp.gmail.com
port:      587
starttls:  true
auth:      login
username:  ваш-адрес@gmail.com
password:  пароль-приложения
```

### Пароль приложения

`auth_password` — не пароль от почты, а отдельный пароль приложения из 16 символов. Создаётся при включённой двухфакторной аутентификации:

```text
https://myaccount.google.com/apppasswords
```

После изменения настроек перезапустите сервер.
